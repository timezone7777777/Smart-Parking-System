# AI Smart Parking System v3 (YOLOv8 + FastAPI + Streamlit)

This is a professional-grade, high-performance license plate recognition and parking management system. Version 3 features a FastAPI backend, an Admin Dashboard, and a Mobile-responsive Driver Website accessible via local network or hotspot. It is optimized for NVIDIA GPUs (like the RTX 3050 Ti) to handle live video streams or webcams in real-time.

---

## Project Structure

* **api.py**: The central logic server. Handles the SQLite database, billing logic with wiggle room, and security checks.
* **main.py**: The Admin Dashboard. Used for live gate monitoring, financial logs, and security/VIP management.
* **driver_app.py**: The lightweight, mobile-optimized website for drivers to check live availability and find their vehicles.
* **detector.py**: Handles the AI logic (YOLOv8 for detection + EasyOCR for reading plates).
* **receipt_gen.py**: Generates high-quality thermal receipt PNGs including vehicle evidence photos.
* **config_manager.py**: Allows editing of parking slots, pricing, and "Free Minutes" wiggle room via the UI.
* **utils.py**: A small helper to check if your GPU is being detected.
* **captured_plates/**: Autogenerated folder storing evidence photos of every vehicle that enters.
* **receipts/**: Autogenerated folder storing digital thermal tickets for exits.

NOTE: 
- parking.db is autogenerated upon first run.
- yolov8n.pt contains the weights for the detection model and is downloaded automatically if missing.

## Prerequisites

* **Python 3.12** or higher installed.
* **NVIDIA GPU Users**: You should have the CUDA Toolkit installed to achieve maximum processing speeds.
* **Non-GPU Users**: If you do not have a dedicated GPU, the code will run on your CPU at a lower frame rate.
* **Windows Environment**: Optimized for Windows Desktop directory and Windows Defender Firewall handling.

## Setting Up a Virtual Environment (Recommended)

It is best to isolate this project to avoid dependency conflicts with other Python projects on your system.

1. **Create the environment**:
   Windows: `python -m venv venv`

2. **Activate the environment**:
   Windows: `.\venv\Scripts\activate`

## Installation and Getting Started

1. Download all project files into a single folder.
2. Open your terminal in that folder and ensure your virtual environment is activated.
3. Install the required libraries using the following command:

```bash
pip install streamlit fastapi uvicorn opencv-python pandas ultralytics easyocr torch torchvision torchaudio pillow qrcode[pil]

Running the System (3-Terminal Workflow)
Since the system is now API-driven, you must run three separate processes in three different terminals:

1. The Backend (API)
In Terminal 1, start the central logic server:

Bash
python api.py
2. The Admin Dashboard (Management)
In Terminal 2, start the main control panel:

Bash
python -m streamlit run main.py --server.port 8501
3. The Driver Website (Mobile Access)
In Terminal 3, start the broadcast for mobile users using your local IP (e.g., 10.18.44.132):

Bash
python -m streamlit run driver_app.py --server.address 0.0.0.0 --server.port 8502
Key Features in v3
API-Driven Architecture: Decoupled logic allows multiple gates and mobile apps to communicate with the same database simultaneously.

Wiggle Room Logic: Configurable Free Minutes ensures drivers are not charged for accidental entries or immediate drop-offs.

Thermal Receipt Generator: Automatically creates a visual PNG ticket for every exit, including the entry photo for security verification.

Driver Mobile Site: Accessible via a dynamic QR code in the Admin Settings. Drivers on the same hotspot can view live floor-by-floor vacancy and find their car's slot number.

Security and VIP Management: Register VIP plates for free parking or Blacklist plates to trigger instant security alerts in the dashboard.

Troubleshooting Local Networking
If the Driver Website is not loading on a mobile device:

Check the URL: Ensure the mobile browser is using http and not https.

Firewall: Ensure Port 8502 (or 8501) is allowed in the Windows Defender Firewall Inbound Rules.

Hotspot: Confirm that both the hosting PC and the mobile device are connected to the same mobile hotspot.
